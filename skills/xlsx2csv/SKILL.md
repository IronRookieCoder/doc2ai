# xlsx2csv

将 `.xlsx` 文件转换为一组 CSV 文件，输出结构清晰、可直接被 AI 或脚本消费。支持单文件或目录批量转换。

## 前置依赖

脚本依赖 `pandas`、`python-calamine` 和 `pyyaml`。如果缺少依赖，先 `pip install` 安装。

## 调用参数

用户调用 skill 时可传入以下参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| `-o <dir>` | 指定 CSV 输出目录（默认 `csv/`） | `/xlsx2csv report.xlsx -o output/` |

## 执行流程

### 步骤 1：确定待转换文件

- 用户可能给出单个文件路径、目录路径或文件列表
- 如果给出目录，脚本自动递归扫描其中所有 `.xlsx` 文件（包含嵌套子目录，跳过 `~$` 开头的临时文件）
- 确认文件列表后开始逐文件处理

### 步骤 2：确定输出目录

- 如果用户传入 `-o <dir>`，使用指定目录
- 否则默认输出到当前工作目录下的 `csv/`

### 步骤 3：运行转换脚本

调用本 skill 自带的转换脚本（默认输出到 `csv/`）：

```bash
python <skill-path>/scripts/convert.py <输入文件.xlsx> -o csv/
```

如用户传入 `-o <dir>`，则使用指定目录：

```bash
python <skill-path>/scripts/convert.py <输入文件.xlsx> -o <用户指定目录>
```

批量转换时传入目录路径：

```bash
python <skill-path>/scripts/convert.py <目录路径> -o csv/
```

`--config`：指定配置文件路径，默认使用 skill 目录下的 `config.yaml`，通常不需要传。

### 步骤 4：输出转换摘要

所有文件处理完毕后，向用户汇总：
- 成功/失败计数（批量时）
- 每个文件的工作表数量和输出路径
- 失败文件的错误原因

## 输出结构

假设输入文件为 `报表.xlsx`，包含工作表"营收"、"成本"、"利润"：

```
csv/
└── 报表/
    ├── 报表.csv          ← 索引文件：汇总各工作表的元信息
    ├── 营收.csv          ← 工作表"营收"的完整数据
    ├── 成本.csv          ← 工作表"成本"的完整数据
    └── 利润.csv          ← 工作表"利润"的完整数据
```

目录批量转换会保留输入目录的相对层级，避免不同子目录中的同名文件互相覆盖：

```
data/
└── 华东/
    └── 报表.xlsx

csv/
└── 华东/
    └── 报表/
        ├── 报表.csv
        └── 营收.csv
```

命名规则：
- 输出目录名 = 原文件名（去扩展名）
- 索引文件名 = 原文件名.csv
- 子表文件名 = 原工作表名.csv

## 索引文件格式

索引 CSV 只包含核心元信息，方便快速了解整个工作簿的结构：

```csv
工作表顺序,工作表名,导出文件名,有效区域
1,营收,营收.csv,A1:E10
2,成本,成本.csv,A1:D8
3,利润,利润.csv,A1:F12
```

## 转换规则

这些规则的核心目标是**忠实还原原始数据结构**，让下游消费方（AI 或脚本）拿到的是原始布局，而不是经过二次加工的数据：

1. 保留原始网格结构（空白单元格、空白行、空白列都保留）
2. 不补全合并单元格 — 合并区域只在左上角保留值，其余位置留空
3. 不做表头推断 — 不假设第一行是表头
4. 不做语义归一化 — 不修改单元格内容
5. 丢弃样式、图片、图表等非数据内容
6. 隐藏的工作表默认不导出（可通过配置开启）

## 配置

`config.yaml` 存放在 skill 目录内，控制转换行为的细节。这是 skill 内部实现的一部分，不会输出到结果目录。

配置项和默认值见 skill 目录下的 `config.yaml`。通常不需要修改，除非用户有特殊要求（如不导出隐藏 sheet）。

## 容错

- **文件损坏或密码保护**：脚本报错并跳过，不中断批量处理中的其他文件
- **单文件失败不中断批次**：记录错误原因，继续处理下一个文件，最终汇总成功/失败计数
- **工作表名冲突**：多个工作表名 sanitize 后重复时（如 `Sheet/1` 和 `Sheet:1` 都变为 `Sheet_1`），自动追加序号（`Sheet_1_2`）避免覆盖
- **编码**：输出 CSV 使用 UTF-8 with BOM 编码，确保中文在 Excel 中直接打开不乱码
- **中文路径与文件名**：脚本使用绝对 `Path` 处理输入输出，支持中文目录、中文文件名和嵌套子目录

## 注意事项

- 如果工作表名包含文件系统不允许的字符（如 `/`、`\`、`:`），脚本会自动替换为下划线
- 空工作表也会导出（生成空 CSV），并在索引中标记有效区域为空
